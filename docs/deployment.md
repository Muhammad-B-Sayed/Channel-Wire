# Deploying ChannelWire on Render and Vercel

This deployment shape runs the C core and FastAPI gateway together on Render, stores data in Render Postgres, and serves the React dashboard from Vercel.

## Architecture

```text
Browser
  -> Vercel React dashboard
  -> Render FastAPI gateway
  -> local C core inside the Render web service
  -> Render Postgres
```

Only the dashboard and gateway are public. The C core and Postgres stay private to the backend service.

## Render backend

1. Create a Render Blueprint from this repository, or create a Docker web service manually.
2. Use `render.yaml`.
3. Set `CHANNELWIRE_DATABASE_URL` to the internal connection string for your existing Render Postgres database.
4. Keep `CHANNELWIRE_DB_SCHEMA=channelwire`.
5. Set `CHANNELWIRE_CORS_ORIGINS` on `channelwire-api` to your Vercel URL, for example:

```text
https://channelwire.vercel.app
```

`CHANNELWIRE_JWT_SECRET` is generated by Render. `CHANNELWIRE_ENABLE_DEV_TOKEN=0` disables demo tokens in production.

For Vercel preview deployments, keep this Render environment variable so generated `*.vercel.app` URLs can call the gateway:

```text
CHANNELWIRE_CORS_ORIGIN_REGEX=https://.*\.vercel\.app
```

ChannelWire can safely share a single Render Postgres database with another project. The app stores its tables and Alembic migration state under the `channelwire` schema instead of the default `public` schema, so the resulting tables are `channelwire.users`, `channelwire.channels`, `channelwire.memberships`, `channelwire.messages`, and `channelwire.alembic_version`.

If your existing database is already used by another app, do not change that app's tables. Just set ChannelWire's `CHANNELWIRE_DATABASE_URL` to the same database connection string and leave `CHANNELWIRE_DB_SCHEMA=channelwire`.

The Render service uses `deploy/render/Dockerfile`, which builds the C server and runs it next to the FastAPI gateway inside one container.

## Vercel frontend

Import this repository into Vercel and use the root `vercel.json`.

Set these Vercel environment variables:

```text
VITE_GATEWAY_URL=https://your-render-service.onrender.com
VITE_ENABLE_DEV_TOKEN=0
```

The Vite build bakes these into the frontend bundle. After changing them, redeploy the frontend.

## Local Docker

Local Docker Compose still supports Dev Token for demos:

```sh
CHANNELWIRE_POSTGRES_PUBLISHED_PORT=15432 docker compose up --build
```

## Production notes

- Do not expose Postgres publicly.
- Do not expose the C core TCP port publicly.
- Keep `CHANNELWIRE_ENABLE_DEV_TOKEN=0`.
- Use a strong generated `CHANNELWIRE_JWT_SECRET`.
- Restrict `CHANNELWIRE_CORS_ORIGINS` to the exact production Vercel domain.
